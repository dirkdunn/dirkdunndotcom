CrowdVibe

CrowdVibe is an application that uses IBM watson to figure out how people feel about an event

During the 2017 SXSW Hackathon, held in Austin Texas, myself and a few others created an application called "CrowdVibe", that used the Ticketmaster API, Twitter API, and Watson API in conjuction to extract and analyze the emotions of those tweeting at events. Ticketmaster API was used to find an event, the lattitude, longitude, and time of the event were then passed into the Twitter API, where all tweets within a 1 mile radius were accumilated during the time of event. All of the tweets were then concatonated together and ran through the Watson sentiment analysis algorithm to extract the emotion from the tweets, allowing the end user to see how people were feeling at the event. Link to project will be up soon!

Slack Translator

I created a chat translator using the Slack API and Watson

I created an extention to the Slack chat program as a AI lesson for my students at General Assembly. Using the slack API, when the user typed in "/translate" followed by a language of their choice E.G. "/translate german", a slack bot named "Dirk's Polyglot Monkey" would show up on the screen and give the translated version of what they had typed. The server endpoint was built with Python Flask, and used IBM Watsons translation algorithm to translate the sentences.

Roman MD5

A fun encryption library that combines MD5 and acient encryption methods.

This project is a true demonstation of my inner geek. One of my favorite coding topics to tinker with is encryption, in this case the MD5 encryption algorithm, in conjunction with ancient encryption methods such as Poly Alphanumeric encryption, ROT13, and the Enigma Cipher.
